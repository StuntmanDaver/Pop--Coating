## Codebase Patterns

- **Turbopack monorepo middleware shadowing**: Next.js 16 + Turbopack uses the pnpm workspace root as the project base for middleware discovery. It finds `src/proxy.ts` at the monorepo root (the SaaS app's middleware) when building pops-website. Fix: create `apps/pops-website/src/proxy.ts` as a no-op stub to shadow the root's middleware for this workspace package.
- All page content sources from .firecrawl/popsindustrial/pages/<slug>.md
- content/company.ts is the single source of truth for business facts
- Footer is rendered from content/site-globals.json#footer.navigation_columns
- Shipping ZIP locked at 33811 (open question §12.1 unresolved)
- Logo canonical: /images/Pops-no-border.png
- pops-website uses `apps/pops-website/app/` for routes (NOT `src/`) — no src-dir convention here
- pops-website .gitignore must exclude node_modules, .next, *.tsbuildinfo to avoid committing build artifacts
- Tailwind v4 is CSS-first: `@import "tailwindcss"` + `@theme { ... }` in app/globals.css; postcss.config.mjs uses `@tailwindcss/postcss`. NO tailwind.config.ts.
- Design tokens (color-pops-yellow-*, color-ink-*, semantic, font-display/text/mono, radius-*, shadow-1..4) are sourced VERBATIM from docs/design/popsindustrial-design-principles.md §10. Do NOT diverge — those values are paint-chip-locked.
- Tailwind v4 auto-generates utility classes from `@theme` tokens (e.g. `bg-ink-800`, `text-pops-yellow-500`, `font-display`, `rounded-sm`, `shadow-3`) — no need to declare them.
- next/font CSS variables (`--font-display`, `--font-text`) are set on `<html>` via the className spread; they CASCADE-OVERRIDE the static fallback stacks declared in `@theme`. The `font-display` / `font-text` Tailwind utilities resolve `var(--font-*)` at the element, so the next/font self-hosted family wins. Self-hosted assets land at `/_next/static/media/*.woff2` — never the Google Fonts CDN.
- Static assets are in `apps/pops-website/public/` (URL: `/images/<file>`, `/pdfs/<file>`, `/favicon.ico`). Server-side content imports live at `apps/pops-website/content/` (e.g. `content/site-globals.json` for footer / nav columns). NEVER serve content/site-globals.json over HTTP — it's an `import` target only.
- Source of static assets: `.firecrawl/popsindustrial/assets/{images,pdfs}/` — 194 images + 4 PDFs. If a story needs an image, check this directory first; the file is already at `/images/<exact-filename>` after US-004.
- Multi-resolution `favicon.ico` was generated via `magick <png> -define icon:auto-resize=16,32,48,64,128,256 favicon.ico`. Re-run that command if `cropped-favicon.png` is ever updated.
- Components live at `apps/pops-website/components/<area>/<file>.tsx` (kebab-case files, PascalCase named exports). Areas: `layout/`, `marketing/`, `ui/`, `forms/`, `seo/` (per the per-AC notes scattered through US-006..US-049).
- Layout primitives use named exports + `forwardRef` to accept refs and pass through HTML element props (`ComponentPropsWithoutRef<"div">` for Container, `ComponentPropsWithoutRef<"section">` for Section). React 19 still supports forwardRef; use it where ACs explicitly require ref forwarding.
- className merging convention until clsx lands: `className ? \`${base} ${className}\` : base`. shadcn primitives (US-008+) will introduce `cn()` from `lib/utils.ts`; switch to that once available.
- Tailwind v4 ships a default 4px spacing scale, so `py-16` / `md:py-24` (= 64px / 96px) match `--space-16` / `--space-24` from design principles §5.1 without declaring a custom `--spacing-*` namespace in `@theme`.
- Container caps at `max-w-[1280px]` with responsive horizontal padding `px-4 sm:px-6 lg:px-8`. Don't lower the cap; the design contract assumes 1280px on every page (hero, contact form, service grid).
- Eyebrow / micro-label convention (design principles §4.3 + §10): `text-xs` (12px) Inter 600 uppercase with `tracking-[0.04em]`. Default brand tone uses `text-pops-yellow-500`; secondary "ink" tone uses `text-ink-400` — the canonical "secondary text" color (per token comment in §3.2). Uppercase is reserved for micro-labels, button text, and the wordmark — never headings.
- `lib/utils.ts` exports `cn(...inputs)` = `twMerge(clsx(inputs))`. Use it in every component that takes a `className` prop or composes Tailwind class lists. Deps already added: `clsx`, `tailwind-merge`. Replace the `className ? \`${base} ${className}\` : base` template-literal merge in older primitives (Container/Section/EyebrowLabel) when next touched.
- shadcn-style `cva()` from `class-variance-authority` is the convention for components with multiple visual variants (Button, future Card, future Input). Define a single `cvaXxx` recipe with `variants` + `defaultVariants`, expose `VariantProps<typeof recipe>` to props, and merge via `cn(recipe({...}), className)`.
- `asChild` pattern via `@radix-ui/react-slot`: when consumers want the component to project its styles onto a child element (typically `<Link>` or `<a>`), accept `asChild?: boolean` and render `Slot` instead of the native element. Slot forwards props and merges classes onto the immediate child. The native element variant is the default — preserve focus, type attributes, ARIA, etc. on it.
- Focus ring convention: `focus-visible:ring-2 focus-visible:ring-pops-yellow-300 focus-visible:ring-offset-2 focus-visible:ring-offset-ink-800` + `outline-none`. Tailwind v4 implements `ring-*` via `box-shadow`, so the ring color (`pops-yellow-300` = `#FFE067`) appears as a 2px outer halo with a 2px ink-800 offset between it and the element — matches `--shadow-focus` from design principles §10. Use `focus-visible` (not `focus`) to avoid showing the ring on mouse click.
- Loading-state width preservation: render the original children invisibly (`aria-hidden + invisible`) so the layout box keeps its measured width, then absolutely-position the spinner in `absolute inset-0`. This avoids a "snap" when toggling `isLoading` mid-submit. Use `aria-busy={isLoading || undefined}` on the button.
- Internal preview pages live at `apps/pops-website/app/dev/<feature>/page.tsx` (NOT `_dev/` — Next.js App Router treats `_folder` as a private folder that is opted out of routing). The `dev/*` namespace must be excluded from the sitemap (US-042) and `robots.ts`. These are working previews for browser verification of components — keep them around as a reference panel for the design system.

  - The `cva` recipe approach is clean for variant explosions (4 variants × 2 sizes = 8 combinations) but overkill for primitives with 1 visual state (e.g. Container/Section/EyebrowLabel). Use cva when there are ≥2 variants of any axis.
  - Loading-state UX: the `invisible inline-flex items-center gap-2` + absolute spinner trick preserves measured width (browser-verified: 177px) so `<Button isLoading>Submitting request</Button>` doesn't jump as the parent toggles state during a Server Action. This is the pattern to reuse in form submit handlers (US-021, US-031, US-032, US-033 → wire-up in US-036+).
  - The `asChild` Slot pattern intentionally skips the loading UI because consumers using `<Button asChild><Link></Link></Button>` are wrapping a navigation element where there's no async submit. If a future story needs an asChild + loading combination (e.g. signed-out CTA that triggers an async redirect), revisit.
  - Internal preview pages MUST live at `app/dev/<feature>/page.tsx` and NOT `app/_dev/<feature>/page.tsx` — App Router treats `_folder` as a private folder opted out of routing (the page just won't be reachable). When US-042 lands, exclude `/dev/*` from the sitemap and `robots.ts`.
