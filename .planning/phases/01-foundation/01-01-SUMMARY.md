## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Compatibility] eslint@9.39.4 used instead of planned @10.3.0**
- **Found during:** Task 2
- **Issue:** eslint@10.3.0 caused peer dependency conflicts with eslint-config-next@16.2.4. The eslint-config-next package requires `eslint >= 9.0.0`, and v9.39.4 is the latest v9 release that is fully compatible.
- **Fix:** Used eslint@9.39.4 which supports flat config (`eslint.config.js`) identically. ESLint v9 and v10 flat config API is compatible.
- **Impact:** None — all ESLint rules work identically; flat config resolution via eslint.config.js is supported in v9.
- **Files modified:** `package.json`

**2. [Rule 3 - Fix] lint script changed from 'next lint' to 'eslint src'**
- **Found during:** Task 2
- **Issue:** `next lint` invokes the Next.js ESLint runner which uses its own config discovery. With eslint@9 flat config and a custom `eslint.config.js`, using `eslint src` directly is more reliable and avoids double-configuration issues.
- **Fix:** Changed lint script to `eslint src && pnpm madge:check`
- **Impact:** Same lint behavior; ESLint correctly uses `eslint.config.js`.

**3. [Rule 3 - Fix] madge:check guards for src/modules not existing**
- **Found during:** Task 2
- **Issue:** `madge --circular src/modules` fails if `src/modules` doesn't exist yet (it's created in Plan 02+).
- **Fix:** Script guarded: `[ -d src/modules ] && madge --circular src/modules || echo '...'`
- **Impact:** `pnpm lint` passes at all times; circular check activates when modules exist.

**4. [Cleanup] Removed eslint.config.mjs (pnpm create next-app artifact)**
- **Found during:** Task 2
- **Issue:** `pnpm create next-app` generated an `eslint.config.mjs` alongside our `eslint.config.js`. ESLint correctly prefers `eslint.config.js` but having two configs is confusing.
- **Fix:** Removed `eslint.config.mjs`. Our `eslint.config.js` (CommonJS, required for eslint-config-next require()) is the sole authoritative ESLint config.
